Code Campus - LMS Platform

Code Campus is a comprehensive Learning Management System (LMS) built with React, TypeScript, Node.js, and MongoDB, designed to revolutionize online education through modern web technologies. The platform provides a complete ecosystem for educational content delivery, featuring course management, secure payment processing via SSL Commerz, video content delivery with advanced playback controls, and real-time collaboration tools.

The platform features SSL Commerz payment gateway integration supporting multiple payment methods (bKash, Nagad, credit/debit cards) for secure course purchases, comprehensive admin dashboard for managing courses, users, payments, and analytics, video upload and streaming with advanced playback controls and progress tracking, and live group chat system enabling real-time communication between course members using Socket.io.

Code Campus empowers administrators with complete control over the platform including course creation and editing, user management with role-based access control, payment transaction monitoring, content moderation, enrollment tracking, and comprehensive analytics dashboard. Students can purchase courses securely, access video content with adaptive streaming, track their learning progress, participate in live group discussions, and download course materials.

Developed by Mhr Rony, Code Campus demonstrates expertise in building scalable educational platforms with real-time features, secure payment integration, and advanced media handling. The platform combines modern TypeScript development with robust backend architecture to deliver a seamless learning experience for both students and administrators.

Présentation

MAHITA v2 est une solution d’assistance intelligente conçue pour améliorer l’autonomie des personnes aveugles ou malvoyantes dans leurs déplacements quotidiens.

Composants du système

  • Caméra du smartphone (vision en temps réel)
  • Intelligence artificielle embarquée (reconnaissance, description, détection)
  • Capteur ultrason (mesure de distance et obstacles proches)

Interaction accessible

  • Voix (STT) → l’utilisateur parle
  • Audio (TTS) → l’application répond
  • Feedback haptique (vibration/buzzer) → alertes instantanées

Objectif : voir à la place de l’utilisateur, prévenir les dangers, et guider ses décisions.

Problématique & Contexte

Les personnes non-voyantes affrontent des obstacles imprévisibles (objets à hauteur de tête, obstacles au sol, trous, personnes proches, environnement urbain). Les solutions existantes sont souvent chères, complexes, dépendantes d’internet et peu intuitives. MAHITA v2 transforme le smartphone en assistant visuel vocal et ajoute une sécurité physique via capteur ultrason et vibrations.

Cause ➜ Conséquence

  • Cause : Impossible de voir l’environnement → Conséquence : chocs, stress → Réponse : détection d’obstacles + alertes.
  • Cause : Manque de description → Conséquence : incompréhension de l’espace → Réponse : IA : description de scène + objets.
  • Cause : Besoin d’interaction sans écran → Conséquence : interfaces inutilisables → Réponse : commandes vocales (STT) + réponses audio (TTS).
  • Cause : Obstacles trop proches → Conséquence : réaction tardive → Réponse : ultrason + vibration instantanée.
  • Cause : Batterie faible → Conséquence : arrêt du système → Réponse : gestion batterie + alertes vocales.

Objectifs & Fonctionnalités

Objectifs sociaux

  • Favoriser l’autonomie
  • Réduire la dépendance
  • Améliorer la sécurité et la confiance

Objectifs techniques

  • Détection d’obstacles en temps réel
  • Description visuelle via IA offline
  • Interaction naturelle sans écran
  • Architecture légère et portable

Objectifs UX (Accessibilité)

  • Une commande vocale = une action
  • Feedback immédiat avant collision
  • Aucune lecture visuelle nécessaire

Fonctionnalités principales

A) Assistance visuelle (caméra smartphone)
  • Description de scène (ex. “Rue, voiture, arbre, personne…”)
  • Détection d’objets proches et importants
  • Reconnaissance faciale (“Personne connue” / “inconnue”)
  • Mode exploration: “Qu’y a-t-il devant moi ?”, “Décris la scène”, “Y a-t-il quelqu’un ?”
B) Sécurité & Obstacles (capteur ultrason)
  • Mesure de distance en continu
  • Alerte progressive: loin (aucune), moyen (vibration légère), proche (vibration + buzzer), très proche (alerte + TTS)
  • Détection: obstacle frontal, risque de collision, zone dangereuse
C) Interaction 100% vocale
  • Commandes STT: “Décris devant moi”, “Lis les objets”, “Batterie”, “Active mode sécurité”
  • Réponses TTS: scène, distance, danger, batterie, personne détectée
D) Gestion intelligente de la batterie
  • Annonces: batterie faible, niveau restant, état de charge
  • Objectif: éviter la panne en déplacement
